# Задание: Структура HTML-документа


## Цель
Научиться создавать базовую структуру HTML-документа с семантическими тегами:  
`header`, `main`, `nav`, `section`, `aside`, `footer` и заголовки/параграфы внутри них.

CSS уже подключён и показывает рамки и подписи к тегам, чтобы вы визуально видели, какой тег где находится.


## Подсказки по HTML


### Основные теги документа
- `<header>` - верхняя часть сайта, обычно логотип, название, заголовки.  
- `<main>` - основной контент страницы.  
- `<nav>` - навигация, меню сайта.  
- `<section>` - логический раздел контента.  
- `<aside>` - боковая панель, дополнительная информация.  
- `<footer>` - нижняя часть сайта, подвал, контакты и копирайты.


### Текстовые теги
- `<h1>` - главный заголовок страницы (например название сайта).  
- `<p>` - абзацы текста.  


### Структура страницы
- В `main` можно использовать **сетки или flexbox**, чтобы расположить `nav`, `section` и `aside`.  
- Каждый `<section>` может содержать несколько `<p>` для текста.  
- `<aside>` часто используется для дополнительной информации, меню, ссылок.


## Подсказки по визуализации CSS
- CSS уже подключён через файл `Структура HTML-документа.css`.  
- Каждый тег визуально обведён рамкой и подписан с помощью `::before` для наглядности.  
- Наблюдайте за цветами рамок и подписи тегов — это поможет понять семантику и вложенность.  
- **Не трогайте CSS**, просто используйте подсказки для правильной структуры HTML.  


## Как работать

1. Создайте HTML-страницу с тегами: `header`, `main`, `nav`, `section`, `aside`, `footer`.  
2. В `header` добавьте `<h1>` с названием сайта.  
3. В `main` разместите:
   - `nav` - навигацию,
   - `section` с двумя `<p>` для описаний,
   - `aside` - боковую панель.  
4. В конце добавьте `<footer>` с текстом подвала.  
5. Сравните результат с файлом `solution` после завершения.


## Совет от Сенсея 
- Смотрите на подписи к тегам и рамки - они помогут понять структуру.  
- Сначала расставьте теги правильно, потом проверяйте текст внутри `<p>` и `<h1>`.  
- Используйте HTML семантику, чтобы ваша страница была логичной и понятной.
