## CSS Arena: Challenge "System Recovery"
### Описание задачи
Это практическое испытание на тему каскадности и специфичности CSS-селекторов.
По сюжету, сайт «CSS ARENA» подвергся хакерской атаке. Злоумышленники внедрили вредоносные стили прямо в HTML-код (тег `<style>` и `inline-атрибуты`), которые сломали визуальное отображение карточек.

### Цель: Восстановить оригинальный дизайн, не внося никаких изменений в файл `task1.html`. Все исправления должны быть реализованы исключительно во внешнем файле `task1.css` за счет повышения приоритета селекторов.

## Технологии
`HTML5` (структура с «взломанным» кодом)

`CSS3` (работа с весами селекторов, Flexbox, Filter, Transform)

## Что было исправлено:
Карточка 1 (Color Restore): Перебит вес селектора `div .card-1` (11) с помощью более глубокой вложенности.

Карточка 2 (Shape Stability): Нейтрализовано влияние идентификатора `#bad-boss` (100) путем комбинации `tag#id` (101).

Карточка 3 (Visibility): Устранена прозрачность через объединение классов `.container .card.card-3.`

Карточка 4 (Darkness Escape): Побежден `inline-стиль` (1000) с помощью легального использования флагмана `!important`.

## Теория (Специфичность)
В проекте наглядно продемонстрирована иерархия весов:

- 1000 — Inline стили.

- 100 — ID.

- 10 — Классы / Псевдоклассы.

- 1 — Теги.
